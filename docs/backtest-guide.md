# 백테스트 완전 가이드

> 이 문서는 페르소나 백테스트 앱의 작동 원리와 결과를 올바르게 해석하는 방법을 설명합니다.

---

## 1. 백테스트란?

**백테스트(Backtest)** 는 과거 데이터에 특정 투자 전략을 적용해 *"만약 그때 이 전략을 썼다면 얼마를 벌었을까?"* 를 시뮬레이션하는 기법입니다.

```
과거 주가 데이터 + 매매 전략 규칙 → 가상의 매매 실행 → 성과 측정
```

### 왜 백테스트를 하나요?

실제 돈을 투자하기 전에 전략의 강점과 약점을 파악할 수 있습니다.
- 어떤 시장 환경에서 수익이 났는가?
- 최악의 경우 얼마나 잃었는가?
- 거래가 충분히 자주 발생하는가?

### 백테스트의 한계

| 한계 | 설명 |
|------|------|
| **과거 ≠ 미래** | 과거에 잘 통한 전략이 미래에도 통한다는 보장이 없습니다 |
| **생존 편향** | 현재까지 살아남은 종목만 분석하므로 실제보다 낙관적으로 보일 수 있습니다 |
| **슬리피지 미반영** | 실제 거래에서는 원하는 가격에 체결되지 않을 수 있습니다 |
| **유동성 가정** | 언제든 원하는 수량을 매매할 수 있다고 가정합니다 |

## 2. 이 앱의 백테스트 구조

### 데이터

- **출처**: Yahoo Finance (`yfinance`)
- **봉**: 1시간봉 (1-hour candlestick)
- **기간**: 최근 6개월
- **시장**: 미국 정규장만 포함 (09:30 ~ 16:00 ET, 주말·공휴일 제외)
- **대상 종목**: AAPL, MSFT, GOOGL, AMZN, NVDA, META, TSLA, JPM, V, WMT

> 1시간봉 6개월 ≈ **약 840개 캔들** (130거래일 × 6.5시간)

### 시뮬레이션 흐름

```
1. 과거 6개월 1시간봉 데이터 로드
2. 첫 24봉은 워밍업 구간 (AI형 LSTM 입력 준비)
3. t=24 봉부터 매봉마다:
   a. 페르소나가 현재까지의 데이터를 보고 매수/매도/관망 결정
   b. 결정에 따라 포트폴리오 업데이트 (수수료 0.05% 적용)
   c. 현재 자산 가치 기록 → 수익 곡선 생성
4. 마지막 봉까지 반복
5. 전체 지표 계산
```

### 포트폴리오 규칙

- 시작 자본: **$10,000**
- 동시에 **1개 포지션만** 보유 가능
- 수수료: 매수·매도 각 **0.05%**
- 공매도 없음 (매수만 가능)

## 3. 페르소나별 전략 상세

### 🛡️ 신중함 (Coward)

가장 보수적인 전략. 안전이 확인됐을 때만 진입합니다.

```
매수 조건:
  1. 최근 12봉 이동평균 > 그 이전 12봉 이동평균 (상승 추세)
  2. 현재가 - 이동평균 < ±0.5% (MA 근처에서만 진입)
  → 두 조건 모두 충족 시 가용 자본의 20% 매수

매도 조건:
  - 진입가 대비 +2% 도달 → 익절
  - 진입가 대비 -2% 도달 → 손절
```

**특징**: 거래 빈도 낮음, MDD 작음, 횡보장에서 거래 거의 없음

---

### 🔥 야수의 심장 (Beast)

모멘텀을 쫓는 공격적인 전략. 올라가는 것에 올라탑니다.

```
매수 조건:
  - 직전 3봉 수익률 합계 > +0.5% (3시간 연속 상승 모멘텀)
  → 가용 자본의 80% 매수

매도 조건:
  - 직전 1봉 수익률 < -0.3% → 즉시 전량 손절
```

**특징**: 거래 빈도 높음, MDD 큼, 강한 추세장에서 강함

---

### 🔄 공포에 투자 (Contrarian)

남들이 팔 때 사는 역발상 전략. 급락을 기회로 봅니다.

```
매수 조건:
  - 직전 1봉 수익률 < -1% (1시간 내 급락)
  → 가용 자본의 50% 매수

매도 조건:
  - 진입가 대비 +1% 도달 → 익절
  - 진입가 대비 -1% 도달 → 손절
```

**특징**: 급락 후 반등 시 빠른 수익, 하락 추세가 지속되면 연속 손절 위험

---

### 🤖 초보 예언가 (AI)

LSTM 딥러닝 모델이 다음 봉의 상승 확률을 예측해 매매합니다.

```
매 봉마다:
  1. 최근 24봉 OHLCV 데이터를 LSTM 모델에 입력
  2. 모델이 다음 1시간 상승 확률 p (0~1) 출력

매수 조건:
  - 포지션 없음 AND p > BUY_THRESHOLD
  → 가용 자본의 50% 매수

매도 조건:
  - 포지션 보유 중
  - MIN_HOLD_CANDLES 이상 경과 AND p < SELL_THRESHOLD
  - OR MAX_HOLD_CANDLES 초과 → 강제 청산
```

#### LSTM 모델 입력 피처

| 피처 | 계산 방법 |
|------|-----------|
| 로그 수익률 | `ln(close_t / close_{t-1})` |
| 가격 범위 비율 | `(high - low) / close` |
| 정규화 거래량 | `(log(volume) - mean) / std` |

#### 기본 파라미터 (최적값, 2026-04-28 기준)

| 파라미터 | 값 | 의미 |
|----------|----|------|
| BUY_THRESHOLD | 0.42 | 42% 이상 상승 확률이면 매수 |
| SELL_THRESHOLD | 0.45 | 45% 미만으로 확률이 떨어지면 매도 |
| MIN_HOLD_CANDLES | 2 | 최소 2시간 보유 후 매도 가능 |
| MAX_HOLD_CANDLES | 96 | 최대 96시간(4거래일) 후 강제 청산 |

> 앱에서 슬라이더로 직접 조정 가능합니다.

## 4. 성과 지표 해석

### 수익률 (Total Return %)

```
수익률 = (최종 자산 / 시작 자본 - 1) × 100
```

- **양수**: 수익 발생
- **음수**: 손실 발생
- 비교 기준: Buy & Hold (같은 기간 동안 사서 보유만 했을 때의 수익률)

---

### MDD (Maximum Drawdown, 최대 낙폭)

```
MDD = min((현재 자산 - 직전 고점) / 직전 고점) × 100
```

포트폴리오가 **고점 대비 최대 얼마나 떨어졌는지**를 나타냅니다.

| MDD | 해석 |
|-----|------|
| 0% ~ -5% | 매우 안정적 |
| -5% ~ -15% | 보통 |
| -15% ~ -30% | 변동성 큼, 버티기 어려울 수 있음 |
| -30% 이하 | 고위험, 심리적으로 매우 힘든 구간 |

> **MDD가 -20%라면**: 고점에서 매수한 사람은 자산이 한때 20% 줄어든 것을 견뎌야 했습니다.

---

### 승률 (Win Rate)

```
승률 = (수익 실현 매도 횟수) / (전체 매도 횟수)
```

- 승률이 높다고 반드시 좋은 전략은 아닙니다
- 승률 40%라도 수익 거래의 이익이 손실 거래의 손실보다 크면 전체 수익이 날 수 있습니다

---

### 거래 횟수

- 매수 기준으로 집계합니다
- 너무 적으면: 전략이 거의 작동하지 않는 것
- 너무 많으면: 수수료 부담이 커짐

---

### 평균 보유 시간

- 매수 시점부터 매도 시점까지의 평균 시간(시간 단위)
- 단기 전략: 수 시간, 장기 전략: 수십~수백 시간

---

## 5. 전체 종목 분석

단일 종목 결과는 **운의 영향**이 클 수 있습니다. 10개 종목 전체를 한 번에 돌리면:

- 전략이 특정 종목에 편향됐는지 확인 가능
- 전략의 **일반성**을 검증할 수 있습니다
- 수익 종목 수 / 전체 10개 비율로 전략의 안정성 판단

---

## 6. 전체 비교 모드

구매한 페르소나를 **동일 종목·동일 기간·동일 자본**으로 동시에 실행합니다.

- 같은 출발점에서 전략 간 성과를 공정하게 비교 가능
- 수익 곡선 차트에서 각 전략의 리스크·변동성 패턴 차이를 시각적으로 확인

---

## 7. 결과를 올바르게 읽는 법

### 좋은 전략의 기준 (참고용)

| 지표 | 참고 기준 |
|------|-----------|
| 수익률 | Buy & Hold 수익률 이상 |
| MDD | -15% 이내 |
| 승률 | 40% 이상 |
| 거래 횟수 | 5회 이상 (통계적 의미) |

### 자주 하는 실수

1. **수익률만 보기** — MDD가 크면 실제로 버티기 어렵습니다
2. **1개 종목 결과로 일반화** — 반드시 여러 종목으로 검증하세요
3. **백테스트 결과를 그대로 믿기** — 실제 투자에는 심리·세금·유동성 등 변수가 추가됩니다
4. **파라미터 과최적화(오버피팅)** — 과거 데이터에만 맞춘 파라미터는 미래에 잘 작동하지 않을 수 있습니다

## 8. 용어 정리

| 용어 | 설명 |
|------|------|
| **캔들 / 봉** | 일정 기간의 시가·고가·저가·종가를 하나로 표현한 단위. 이 앱은 1시간봉 사용 |
| **이동평균 (MA)** | 최근 N봉의 종가 평균. 추세 방향 판단에 사용 |
| **모멘텀** | 가격 변화의 속도·방향. 오르는 것이 계속 오르는 경향 |
| **역발상 전략** | 시장의 반대로 행동하는 전략. 과매도 시 매수, 과매수 시 매도 |
| **LSTM** | Long Short-Term Memory. 시계열 데이터 예측에 특화된 딥러닝 모델 |
| **로그 수익률** | `ln(P_t / P_{t-1})`. 수익률을 정규 분포에 가깝게 변환한 값 |
| **수수료 (Fee)** | 거래 시 발생하는 비용. 이 앱은 매수·매도 각 0.05% 적용 |
| **Buy & Hold** | 처음에 사서 끝까지 보유하는 단순 전략. 비교 기준으로 사용 |
| **오버피팅** | 과거 데이터에 지나치게 맞춰 미래 데이터에는 잘 작동하지 않는 현상 |
